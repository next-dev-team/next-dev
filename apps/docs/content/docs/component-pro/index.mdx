## Overview

Component Pro provides advanced layout, page container, form, and table components with Ant Design Pro-like APIs, built on top of `@rnr/rnr-ui` and `@registry` components. These components offer enhanced functionality including:

- **ProLayout**: Application layout with side/top/mix navigation, breadcrumb, and header actions
- **PageContainer**: Standardized page header, breadcrumb, operation area, tabs, and watermark
- **ProForm**: Enhanced form component with built-in validation, layout options, and submit handlers
- **ProTable**: Advanced table component with data fetching, pagination, search, and filtering

## Components

### [ProForm](/docs/component-pro/pro-form)

An enhanced form component with built-in validation, layout options, and submit handlers. Supports various field types including text, select, textarea, checkbox, radio, and number inputs.

## Features

- ✅ **Same API as Ant Design Pro** - Familiar API for developers coming from Ant Design Pro
- ✅ **Built on @rnr/rnr-ui** - Uses only `@registry` and `@rnr/rnr-ui` dependencies
- ✅ **TypeScript Support** - Full TypeScript support with proper types
- ✅ **Validation** - Built-in form validation with `rc-field-form`
- ✅ **Pagination** - Built-in pagination support for tables
- ✅ **Search & Filter** - Advanced search and filtering capabilities
- ✅ **Customizable** - Highly customizable with props and render functions

## Getting Started

Install the required dependencies:

```bash
npm install @rnr/rnr-ui rc-field-form @rn-primitives/table
```

Then import and use the components:

```tsx
import { ProForm, ProFormText } from '@/components/ui/pro-form';
import { ProTable } from '@/components/ui/pro-table';
```
